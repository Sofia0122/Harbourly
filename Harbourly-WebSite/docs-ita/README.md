# docs-ita

Documentazione di progetto in italiano per architettura, linee guida UI, routing, integrazione API e workflow di sviluppo.

Mantieni qui le decisioni implementative quando hanno impatto su backend futuro, Figma o design system.

## Contesto Per Sessioni Future

Usa questi file quando vuoi dare contesto rapido a Codex senza far rileggere tutto il progetto:

- `context/project-brief.md`: riassunto operativo del progetto, stack, architettura e regole.
- `context/project-map.md`: mappa delle cartelle e responsabilita.
- `context/session-start.md`: prompt pronto da usare a inizio sessione.
- `context/iteration-rules.md`: regole da seguire a ogni iterazione, inclusa la manutenzione della documentazione.
- `folders/*.md`: schede sintetiche per le cartelle principali di `src`.

Ordine consigliato: `context/project-brief.md`, `context/project-map.md`, `context/iteration-rules.md`, poi la scheda `folders/` della cartella che vuoi modificare.

# Regole Per Ogni Iterazione

Queste sono le istruzioni chiave da dare a Codex all'inizio di ogni iterazione sul progetto.

- Aggiorna i file di context e documentazione quando chiudiamo un argomento importante: bug risolto, feature aggiunta, decisione architetturale presa, cambio di flusso, modifica rilevante a cartelle o convenzioni.
- Non aggiornare la documentazione a ogni singolo messaggio: fallo quando il lavoro produce una conclusione stabile che sara utile nelle sessioni future.
- Se durante una sessione emergono piu modifiche importanti, raccoglile e proponi un aggiornamento documentale a fine blocco di lavoro.
- Se non e chiaro se una modifica merita documentazione, chiedi conferma invece di creare rumore nei file.
- Prima di chiudere una feature o un problema, controlla se vanno aggiornati `docs-ita/context`, `docs-ita/folders`, i README locali o i documenti specifici in `docs-ita`.

# docs

Project documentation for architecture, UI guidance, routing, API integration and workflow.

Keep implementation decisions here when they affect future backend, Figma or design-system work.

## Future Session Context

Use these files when you want to give Codex fast context without rereading the whole project:

- `context/project-brief.md`: operational project summary, stack, architecture and rules.
- `context/project-map.md`: folder map and responsibilities.
- `context/session-start.md`: ready-to-use session prompt.
- `context/iteration-rules.md`: rules to follow every iteration, including documentation maintenance.
- `folders/*.md`: concise notes for the main `src` folders.

Recommended order: `context/project-brief.md`, `context/project-map.md`, `context/iteration-rules.md`, then the relevant `folders/` note for the area being changed.
